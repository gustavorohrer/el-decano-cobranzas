# OpenSpec Proposal Workflow
Use this rule when the user wants to start a new feature, a big architectural change, or a significant refactor.
## Instructions
1. Refer to `docs/workflows/openspec/proposal.md` for the technical workflow.
2. For a practical step-by-step on how to add a feature in this project, consult `docs/workflows/sdd-step-by-step.md`.
3. Scaffold the proposal in `openspec/changes/<change-id>/` before writing any application code.
This workflow can be invoked using @openspec-proposal.
